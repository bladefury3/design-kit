Phase 1: Setup

Catalog what you have

One-time extraction. Caches your design system locally so every skill runs fast.

/setup-tokens
Colors, spacing, radii, typography from Figma variables
/setup-components
Every component with variant keys, props, and smart defaults
/setup-relationships
How components depend on each other
/setup-icons
Icon names mapped to keys with search tags
Phase 2: Create

The design loop

Describe what you need. Get a plan. Build it in Figma. All with your library components and tokens.

/plan → /build
Design a screen. Maps your brief to library components, tokens, and layout. Then builds it.
/flow
Multi-screen flows. Connected sequences with annotations between screens.
/brainstorm
3-5 design variations using SCAMPER + Jobs-to-be-Done. Pick the best direction.
/plan-component → /build-component
New components. Duplicate check, variant matrix, props, tokens. Builds as a component set.
/responsive
Desktop to tablet + mobile. Content choreography and touch-first adaptation.
Phase 3: Review

Check and fix

Automated quality checks. Scores with evidence. Findings posted as Figma comments.

/audit
Nielsen heuristics, Gestalt, cognitive load, token compliance. Weighted score out of 10.
/stress-test
Break it before users do. Long names, huge numbers, empty states, RTL text.
/review-component
9 quality dimensions: variants, tokens, accessibility, naming, props, relationships.
/revise
Targeted fixes from audit or review. Modifies frames without rebuilding.
/diff
What changed in the design system since your last extraction.
Phase 4: Handoff

Ship to engineering

Annotate designs with specs that Dev Mode doesn't show. Token names, interaction states, focus order.

/handoff
Developer-ready specs. Token names, conditional logic, content rules, animation specs. Creates a spec section next to your design.
/handoff-ai
Optimize for AI tools. Enrich descriptions, standardize naming, add structured annotations for MCP consumption.
Why this works

Not just AI generation

Design Kit does

  • Uses YOUR library components (not generic shapes)
  • Binds every value to YOUR tokens
  • Checks for duplicates before creating
  • Applies typicalOverrides automatically
  • Posts findings as Figma comments
  • Asks before making decisions
  • Scores with evidence, not opinions

Generic AI doesn't

  • Invents components that already exist
  • Hardcodes colors and spacing
  • Creates duplicates without checking
  • Leaves icon placeholders visible
  • Outputs JSON files nobody reads
  • Makes assumptions without asking
  • Gives vague "looks good" feedback
